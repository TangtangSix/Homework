三、问题

 Problem 

1、已解决问题：
    (1）MDB样式无法正常使用。
        解决方法：
        https://mdbootstrap.com/docs/b5/angular/getting-started/installation/
    
    (2) Material Icon无法正常显示图标。
        解决方法：在项目的index.html文件中的head标签中加入
        <link rel="stylesheet" href="https://fonts.googleapis.com/icon?family=Material+Icons">
    
    (3) 再使用Material组件库出现
        
        错误提示,一般都是app.module.ts中缺少对对应模块的import,此类错误只需要在
        https://material.angular.cn/components/paginator/overview
        中找到对应模块,把API中import语句复制到app.module.ts中,并在import中导入即可。
        类似错误都可尝试此方法解决。

    (4) 在hero-list组件中进行删除操作后mat-table中没有同步操作
        解决方法: 在删除函数中再执行一次对dataSource和paginator的重新赋值。
    
2、未解决问题
        (1)在使用mat-table中，如果在每一行中放入按钮同时对mat-row标签添加routerLink事件后，
        当点击按钮后同时触发按钮对应事件和routerLink绑定事件。
            现在采取暂时放弃routerLink事件，额外添加一个按钮事件来代替routerLink对应事件。
        
        (2) 当mat-table中的数据来源于ajax时,mat-sort就失去作用。
    
        (3) 当使用路由标签router-outlet的时候,input和output无法在父组件和子组件直接进行
        相关变量操作。

        (4) 无法在标签mat-error使用自定义函数来决定是否显示

四、总结

 Summarize 

    通过此次angular项目实践，不仅使我对angular框架有了更深一步的了解，同时还增强了自我学习能力。
相比较于传统的通过html+css+js的模式，通过使用框架可以大大节省我们项目开发的时间以及开发难度。
    当然，前提是我们必须能够熟练的掌握并且使用对应框架的各种工具。虽然前期学习angular的时候很痛苦
，但是当我能够开始驾驭它的时候，才能真正体验它的便捷和强大。
